# Технічне завдання
## Система управління бібліотекою

### 1. Загальні відомості

#### 1.1 Повна назва системи
Система управління бібліотекою (Library Management System)

#### 1.2 Призначення системи
Автоматизація процесів управління бібліотечним фондом, обліку читачів, видачі та повернення книг.

#### 1.3 Цілі розробки
- Спростити процес управління книжковим фондом
- Автоматизувати облік читачів
- Забезпечити швидкий пошук книг
- Відстежувати історію видачі та повернення книг

### 2. Функціональні вимоги

#### 2.1 Управління книгами

**2.1.1 Додавання книг**
- Система повинна дозволяти додавати нові книги до бібліотеки
- Обов'язкові поля: назва книги, автор
- Кожна книга автоматично отримує статус "доступна"

**2.1.2 Видалення книг**
- Система повинна дозволяти видаляти книги з бази
- Видалення можливе тільки для доступних книг

**2.1.3 Редагування інформації про книгу**
- Можливість змінювати назву книги
- Можливість змінювати автора
- Можливість змінювати статус доступності

**2.1.4 Пошук книг**
- Пошук за автором
- Пошук за назвою
- Пошук за категорією
- Система повинна повертати список знайдених книг

#### 2.2 Управління читачами

**2.2.1 Реєстрація читачів**
- Система повинна дозволяти реєструвати нових читачів
- Обов'язкові поля: ім'я, email
- Валідація email адреси

**2.2.2 Редагування інформації читачів**
- Можливість змінювати ім'я читача
- Можливість змінювати контактні дані

**2.2.3 Перегляд інформації про читача**
- Відображення особистих даних
- Список взятих книг
- Історія взяття/повернення книг

#### 2.3 Видача та повернення книг

**2.3.1 Видача книги**
- Система повинна реєструвати дату видачі
- Книга автоматично отримує статус "недоступна"
- Книга додається до списку книг читача

**2.3.2 Повернення книги**
- Система повинна реєструвати дату повернення
- Книга автоматично отримує статус "доступна"
- Оновлення історії читача

#### 2.4 Звітність

**2.4.1 Інформація про книги читача**
- Список всіх взятих книг
- Дати взяття книг
- Дати повернення (якщо книга повернена)
- Статус кожної книги

**2.4.2 Перегляд книжкового фонду**
- Список всіх книг в бібліотеці
- Інформація про доступність
- Інформація про читачів, які взяли книги

### 3. Технічні вимоги

#### 3.1 Програмне забезпечення
- **Мова програмування:** C#
- **Платформа:** .NET 6.0 або вище
- **IDE:** Visual Studio 2022
- **Система контролю версій:** Git
- **Хостинг:** GitHub

#### 3.2 Архітектура системи

**3.2.1 Структура проектів**
```
LibraryManagementSystem/
├── Library.Core/          # Інтерфейси
│   ├── IBook.cs
│   ├── IReader.cs
│   └── ILibrarySystem.cs
├── Library/               # Реалізація
│   ├── Book.cs
│   ├── Reader.cs
│   ├── LibrarySystem.cs
│   └── Program.cs
└── LibraryTest2/          # Тести
    └── UnitTest1.cs
```

**3.2.2 Інтерфейси**

**IBook**
- GetTitle(): string
- GetAuthor(): string
- IsAvailable(): bool
- UpdateAvailability(bool): void
- GetDateTaken(): DateTime
- GetDateBack(): DateTime
- SetDateTaken(DateTime): void
- SetDateBack(DateTime): void

**IReader**
- Register(string name, string email): void
- Change(string name, string email): void
- BorrowBook(IBook book): void
- ReturnBook(IBook book): void
- InfoReader(): string
- AboutBookOfReader(): string

**ILibrarySystem**
- AddBook(string title, string author): void
- RemoveBook(IBook book): void
- SearchBook(string query): List<IBook>
- CheckAvailability(IBook book): bool
- AddReader(IReader reader): void
- GetBooks(): List<IBook>
- GetReaders(): List<IReader>

#### 3.3 Вимоги до якості

**3.3.1 Тестування**
- Покриття Unit-тестами не менше 80%
- Всі публічні методи повинні бути покриті тестами
- Тестування валідації вхідних даних

**3.3.2 Обробка помилок**
- Валідація всіх вхідних параметрів
- Генерація виключень при невалідних даних
- Обробка помилок в консольному інтерфейсі

**3.3.3 Безпека даних**
- Валідація email адрес
- Перевірка на NULL та порожні рядки
- Захист від некоректних операцій

#### 3.4 Інтерфейс користувача

**3.4.1 Консольний інтерфейс**
- Меню бібліотекаря для управління книгами
- Меню читача для роботи з книгами
- Меню реєстрації користувачів
- Зрозумілі повідомлення про помилки
- Підтвердження виконання операцій
